다음은 내가 만들고 있는 웹서비스 **Flow Map**의 제품 방향과 UX 설계 원칙이다.

이 내용을 기준으로 이후 기능 구현 및 UI 설계를 진행해줘.

## **서비스 목적**

이 서비스의 핵심 목표는 단순히 일을 "빨리 시작하게 만드는 것"이 아니라,

**사용자가 생각을 외부로 꺼내고, 구조화하며, 흐름을 유지하도록 돕는 사고 보조 도구**를 만드는 것이다.

특히 다음 문제를 해결하는 것이 목적이다:

- 머릿속에는 아이디어가 많은데 정리가 되지 않아 막막함을 느끼는 상태
- 프로젝트나 학습을 진행하다가 전체 흐름을 자주 잃는 문제
- 구조가 너무 딱딱하면 창의성이 막히고, 너무 자유로우면 길을 잃는 문제

이 서비스는 **정형성과 자율성 사이의 균형**을 잡아주는 구조를 제공해야 한다.

---

## **핵심 컨셉**

사용자가 떠오르는 생각을 자유롭게 쏟아내면,

AI는 이를 정답처럼 확정하지 않고 **초안 수준의 구조로 정리**해주고,

사용자는 그 구조를 **직접 편집하면서 사고를 발전시키는 웹 기반 브레인스토밍 도구**다.

---

## **기본 UX 흐름**

1. 사용자가 주제 또는 프로젝트를 입력한다
2. 화면 한쪽에서 자유기입 영역에 생각을 제한 없이 작성한다 (브레인 덤프)
3. 사용자가 "정리하기"를 누르면 AI가 입력된 내용을 바탕으로 마인드맵 초안을 생성한다
4. 사용자는 생성된 구조를 보면서 생각을 추가·수정하며 사고를 계속 발전시킨다
5. 이후에도 다시 들어와 현재 구조를 보며 전체 흐름을 상기하고 이어서 작업할 수 있어야 한다

---

## **AI 역할 설계 원칙 (과개입 금지)**

AI는 사용자의 사고를 "대신 결정하는 존재"가 아니다.

AI의 역할은 **사고 보조 및 초안 제안**에 한정된다.

### **기본 모드 (항상 활성)**

AI는 다음과 같은 방식으로 사고를 촉진한다:

- 생각을 확장할 수 있는 질문 제안
- 서로 연결될 수 있는 아이디어 간의 연결 가능성 제안 (단정 금지)
- 현재 구조에서 빠져 있는 관점 제안

이 단계에서는 AI가 구조를 확정하지 않는다.

### **옵션 모드 (사용자 요청 시에만)**

사용자가 명시적으로 "정리하기"를 선택했을 때만,

AI는 입력된 생각을 바탕으로 **마인드맵 형태의 '초안(Draft)' 구조**를 생성할 수 있다.

- 이 결과는 정답이 아니라 Draft로 취급된다
- AI는 최종 구조를 확정하지 않는다

### **금지 사항**

- AI가 사용자 원문을 삭제하거나 대체하지 않는다
- AI가 구조를 "최종 결정"하지 않는다

---

## **사용자 편집권 (최상위 원칙)**

마인드맵 구조의 최종 소유권과 결정권은 항상 사용자에게 있다.

- AI가 제안한 질문, 연결, 초안 구조는 모두 사용자가 수정 가능한 상태여야 한다
- 사용자의 편집 행위(노드 수정, 이름 변경, 연결 재구성 등)가 실제 구조를 확정하는 기준이 된다
- AI는 사용자의 수정 결과를 자동으로 덮어쓰지 않는다
- 사용자가 다시 정리를 요청했을 때만 새로운 초안을 제안할 수 있다

즉, AI는 "제안자", 사용자는 "편집자이자 최종 결정자"다.

---

## **화면 구조 방향**

- 자유기입 영역과 마인드맵 시각화 영역은 명확히 분리되어야 한다
- 시각화의 중심은 "리스트"가 아니라 "흐름(Flow)"이다
- 현재 선택된 노드 또는 단계는 선명하게 강조하고, 나머지는 약하게 표현하되 완전히 숨기지 않는다
- 처음부터 끝까지 고정된 계획처럼 보이지 않도록, 먼 단계일수록 덜 확정적으로 표현한다
- 디자인은 밝고 가독성 중심으로 하며, 과도한 장식적 연출이나 어두운 톤은 지양한다

---

## **현재 기술 상태 (참고용)**

- 현재 프로토타입은 단일 HTML + 바닐라 JS + Tailwind CDN 기반이다
- 데이터는 Gemini API 직접 호출 또는 로컬 mock data 폴백 구조로 되어 있다
- Supabase 연동은 필수가 아니며, MVP 범위와 안정성을 기준으로 선택한다

---

이 원칙을 깨지 않는 범위 내에서 UI, 인터랙션, 구조 설계를 진행해줘.

특히 **AI가 구조를 '결정'하지 않도록** 하고, **사용자 편집권이 항상 위에 오도록** 구현해줘.
